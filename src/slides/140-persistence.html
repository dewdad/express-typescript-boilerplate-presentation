Data Access Object (DAO) Pattern or Repository



  • All database access in the system is made through a DAO to achieve encapsulation.
  • Each DAO instance is responsible for one domain object or entity.
  • The DAO is responsible for CRUD operation on the domain object.
  • The DAO may allow queries based on criteria other than the primary key, which returns a collection of the domain object.
  • The DAO is not responsible for handling transactions, sessions or connections - These are handled outside the DAO to achieve flexibility.
